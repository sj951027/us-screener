## 0. 한 줄 요약

한국판에서 이식하는 건 **모델이 아니라 규율과 뼈대**다. 데이터 레이어는 새로 만들고,
전략은 쌓인 데이터 위에서 wu_factor_scan 방식으로 발견 → 사전등록 → OOS 판정한다.

## 1. 이식하는 규율 (dh-q7m3k에서 검증된 것 — 그대로 계승)

1. **관측 우선**: 팩터는 가중치 0으로 먼저 적재 → forward 검증 후에만 승격.
2. **스펙 동결 + 사전등록**(PREREGISTER_*.md) → **OOS 40거래일** + 부트스트랩 CI +
   Bonferroni + 방향일치. "채택 안 함"도 정당한 결론. 경계값 = 기움.
3. **포인트-인-타임 정직성**: 시총 = 당시 가격×주식수, 팩터는 기준일 이전 데이터만.
4. **매직넘버 금지**: 근거 없는 하드코딩 파라미터 금지 — 플래그→IC→가중.
5. **수집과 계산 분리**(§21-2): raw는 최대한 모으고, 점수는 그 위에서 자유 탐색.
6. **자동매매 금지**: 조회·표시·알림까지만.
7. 운영 교훈 (2026-07 실측): run_id=거래일 앵커, 부분/이중실행 **게이트**, 완전성 게이트,
   cleanup 보존정책+백업을 **day 1부터**, 백업은 파일복사 금지·**sqlite 백업 API**(핫카피
   malformed 실측), 밸류 스냅샷은 **처음부터 DB 적재**(한국판 P0 교훈 — 버리지 말 것),
   상폐·편출입 이벤트 로그 day 1부터(생존편향 전진 차단), 보이지 않는 문자(BOM) 코드 금지.

## 2. 1단계 — 데이터 토대 (구축 시작점, 판정 후 첫 작업)

**us_ohlcv.db** (repo 밖 `../us-screener-data/`, 한국판과 동일 격리):

| 테이블 | 내용 | 소스 | 비고 |
|---|---|---|---|
| daily_ohlcv | **전체 상장(NYSE+NASDAQ ~8천)** OHLCV(조정+비조정)·거래량·주식수 | yfinance | 3년 백필 + 매일 증분. 분할·배당 조정계수 별도 컬럼 |
| valuation_daily | trailingPE·PB·PS·배당수익률·시총 | yfinance info | **day 1부터 일일 스냅샷 적재**(P0 교훈) |
| index_membership | S&P500·NDX 구성 종목 일일 기록 | 위키피디아/공식 | 'big 플래그'용 + **편출입이 곧 생존편향** — 매일 diff 로그 |
| universe_events | 상폐·티커변경·SUSPENDED | 유니버스 diff | 한국판 universe_events 이식 |
| market_daily | SPX·NDX·VIX·DXY·미10Y·USDKRW | yfinance | 노출/레짐 레이어 기반(P2) |
| earnings_dates | 실적 발표 예정일 | yfinance | **미국 특유 리스크** — 어닝 갭. 표시·가드용 |

**전체 수집 ↔ 분석 가드 분리** (한국판 whole_score 가드의 미국판):
- 수집은 전체(~8천, OTC/핑크시트는 제외 — 데이터 품질·조작 위험이 이득을 압도).
- 분석·표시 가드: 주가 ≥ $5(페니주 컷) · 일평균 거래대금 하한(분포 실측 후 결정, 매직넘버
  금지) · 상장 후 최소 거래일 · 거래정지/무변동 컷. 가드는 팩터가 아니라 유니버스 정의.
- 규모 실측 참고: 한국판 2,648종목 3년 백필 = 10분 → 8천 종목 ≈ 30~40분 + 실패 재시도
  (1회성). 일일 증분은 문제 없음. yfinance rate limit 걸리면 배치 분할·야간 재시도.

**미국 특이사항 (한국판과 다른 것)**:
- **상하한가 없음** → ±30% 점프 가드 무의미. 대신 분할 조정계수로 가격 점프 구분,
  전방 |일수익| 컷은 분포 실측 후 결정(매직넘버 금지 — 예: 상위 0.1% 분위 기반).
- **일별 기관 수급 없음**: 한국 KIS급 데이터 부재. 13F는 분기+45일 지연, 공매도 잔고는
  FINRA 2주 지연 — 있는 그대로 지연 명시하고 적재(상상 금지). options 미수집(v2 검토).
- **실행 시간 (한국판과 완전히 다름 — 별도 스케줄)**:
  미국 정규장 마감 = 한국 **새벽 05:00**(서머타임 3~11월) / **06:00**(겨울).
  yfinance 일봉은 마감 직후엔 정정·조정이 덜 끝난 경우가 있어 1~2시간 여유를 둔다.
  → **최적 실행 = 한국 아침 07:30~08:30** (여름 기준 마감+2.5시간, 겨울에도 +1.5시간).
  이러면 신호 산출 → 그날 밤 미국 개장(한국 22:30/23:30)까지 **14시간+ 여유** — 신호 보고
  주문 준비할 시간이 한국판(당일 저녁 산출→다음날 아침 개장)보다 오히려 넉넉하다.
  한국판 .bat은 저녁, 미국판은 아침 — 서로 안 겹치고 PC 부하도 분산. 아침 실행은
  Windows 작업 스케줄러 등록 권장(수동으로 아침마다 누르는 건 지속 어려움).
  ⚠️ 서머타임 전환일(3월·11월) 마감시각 1시간 이동 — 스케줄을 07:30 고정하면 겨울에도
  마감+1.5h 라 안전. run_id 는 **미국 거래일**(ET 기준) 앵커 — 한국 날짜 아님(요일 어긋남 주의).
- 어닝 시즌 변동성, 배당·분할 빈번 → 조정가 일관성 검증을 무결성 체크에 포함.
- 세제·환율은 스크리너 범위 밖(표시만 USD, 참고용 USDKRW 병기 검토).

**yfinance 운영 가드**: rate limit 대비 증분·재시도·idempotent, 실패 티커 pending 기록
(비치명), 주 1회 무결성 체크(NULL/음수/조정 모순). 유료 API 전환 여지는 인터페이스 분리로 확보.

## 3. 2단계 — 팩터 관측 스캔 (데이터 1~2개월 축적 + 3년 백필 후)

wu_factor_scan.py 를 이식해 동일 프로토콜(h20 주지표·블록부트 CI·연도 안정성·top-decile
수익 병행)로 전수 스캔. 후보 축(전부 가설 — 스캔이 결정):
- 가격: mom12(12-1)·nh252(신고가근접)·**upratio63(한국서 최강 후보)**·lv63·max5(복권성)·FIP
- 미국 특유: 어닝 후 드리프트(PEAD — earnings_dates 필요)·배당수익률·자사주(shares 감소율)
- 밸류: E/P·B/P (valuation_daily 축적분 — 처음부터 쌓으므로 몇 달 뒤 가능)
- **주의**: 미국은 팩터 문헌이 방대 = 데이터마이닝 위험도 큼. 문헌 팩터도 자체 데이터로
  재검증 전엔 가설. 한국판 교훈: 정교한 모델도 IC≈0일 수 있다.

## 4c. 추가 고민 지점 (설계 확정 전 열린 문제)

1. **티커 재사용**: 미국은 상폐 티커가 새 회사에 재배정되는 일이 흔함 — symbol 만으로는
   시계열이 오염될 수 있음. listing_events 의 DISAPPEARED→NEW 시퀀스로 감지하고,
   재사용 티커는 백테스트에서 분리 취급(구현 시 명세).
2. **yfinance = 비공식 스크레이퍼**: 야후가 구조 바꾸면 언제든 깨짐. 수집 인터페이스를
   분리해 유료 API 로 갈아끼울 수 있게(이미 §2 반영). 씨앗 수집기의 NASDAQ Trader
   심볼 디렉터리는 공식 소스라 이 리스크 없음.
3. **벤치마크 사전 명기**(한국판 §7 교훈 — EW −15% vs CW +174% 괴리): 미국도 EW vs
   SPX(시총가중) 괴리 큼. 시장초과의 기준을 PREREGISTER 에 EW(가드 유니버스)로 고정,
   SPX 참고 병기.
4. **어닝 게이트**: 실적 발표 직전 종목의 신호 처리(제외? 플래그?) — 팩터 스캔 때
   어닝 전후 수익 분포를 실측한 뒤 결정(매직넘버 금지).
5. 규모: 8천 종목 × 3년 일봉 ≈ 600만 행(~1GB) — 한국판 270MB의 3~4배. repo 밖 격리
   동일, 백업은 core(백필불가만) 우선 검토.
